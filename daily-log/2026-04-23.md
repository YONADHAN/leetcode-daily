## Problems Solved

1. 374. Guess Number Higher or Lower (Easy)

---

## Approach Summary

### Solution 1 (Initial Mistake)

* Used Binary Search idea
* But calculated `mid` incorrectly:

  ```js
  mid = Math.floor((left + (right - left)) / 2)
  ```
* This simplified to `right / 2` → ignored `left`

❌ Issues:

* Search space not reduced properly
* Could lead to incorrect results or infinite loop

---

### Solution 2 (Final - Passed)

* Applied correct Binary Search
* Used:

  ```js
  mid = Math.floor((left + right) / 2)
  ```
* Called `guess(mid)` API
* Adjusted range based on response:

  * `-1` → move left (`right = mid - 1`)
  * `1` → move right (`left = mid + 1`)
  * `0` → return `mid`

---

## Key Learnings

* Binary Search must always use **correct mid calculation**
* Small mistake in formula can completely break logic
* Learned how to use **external API inside Binary Search**
* Reinforced concept of **search space reduction**
* Understood pattern:
  → “move left” vs “move right” based on condition
